# 주보라 주문 관리 Apps Script – 설치 가이드

---

## ① 구글 폼 + 스프레드시트 연결 확인

구글 폼에서 **응답 → 스프레드시트 연결**이 되어 있어야 합니다.

폼 응답 시트를 열면 타임스탬프 + 16개 응답 컬럼이 자동으로 생성됩니다.

---

## ② Apps Script 열기

스프레드시트에서:

```
확장 프로그램 → Apps Script
```

---

## ③ 스크립트 붙여넣기

1. `코드.gs` 파일의 기존 내용을 **모두 삭제**
2. `주보라_주문관리.gs` 파일 내용 전체를 **복사해서 붙여넣기**
3. 상단 CONFIG 블록에서 이메일 주소 확인:

```js
const CONFIG = {
  OWNER_EMAIL: 'mm1895@naver.com',  // ← 알림 받을 이메일
  ...
};
```

4. 💾 **저장** (Ctrl+S)

---

## ④ 컬럼 순서 확인 (중요!)

`COLS` 상수가 **구글 폼 응답 순서**와 일치해야 합니다.

폼 응답 시트의 B열부터 확인:

| 열  | 항목          | COLS 값 |
|-----|---------------|---------|
| A   | 타임스탬프     | 0       |
| B   | 주문자 이름    | 1       |
| C   | 연락처         | 2       |
| D   | 이메일         | 3       |
| E   | 배송지 주소    | 4       |
| F   | 제품 종류      | 5       |
| G   | 크기           | 6       |
| H   | 재질           | 7       |
| I   | 마감           | 8       |
| J   | 수량           | 9       |
| K   | 양면 여부      | 10      |
| L   | 메인 문구      | 11      |
| M   | 서브 문구      | 12      |
| N   | 글꼴           | 13      |
| O   | 예상 금액      | 14      |
| P   | 요청사항       | 15      |
| Q   | 희망 납기일    | 16      |

순서가 다르면 `COLS` 오브젝트의 숫자를 맞게 수정하세요.

## ⑤ 최초 1회 실행

스프레드시트를 새로고침하면 상단에 **🏷 주보라 관리** 메뉴가 생깁니다.

### 1단계: 시트 초기화
```
🏷 주보라 관리 → ① 시트 초기화 (최초 1회)
```
- 주문목록 시트 헤더 서식 적용
- 상태 드롭다운 추가
- 대시보드 시트 생성

### 2단계: 트리거 등록
```
🏷 주보라 관리 → ② 트리거 등록 (최초 1회)
```
- 권한 승인 팝업이 뜨면 **허용** 클릭
- 세 가지 트리거 자동 등록

### 3단계: 이메일 테스트
```
🏷 주보라 관리 → 테스트: 샘플 알림 이메일 발송
```
- `mm1895@naver.com` 으로 테스트 메일 도착 확인

## ⑥ 동작 흐름

```
고객이 주문 폼 제출
       ↓
onFormSubmit() 실행
       ↓
┌─────────────────────────────┐
│ 1. 주문번호 자동 생성        │
│    ORD-20260413-001         │
│ 2. 상태 = "접수" 설정        │
│ 3. 행 배경색 → 연노랑        │
│ 4. 사업자 이메일 알림 발송   │
│ 5. 대시보드 갱신             │
└─────────────────────────────┘

담당자가 상태 변경 (예: "시안발송")
       ↓
onEdit() 실행
       ↓
┌─────────────────────────────┐
│ 1. 행 배경색 → 연파랑        │
│ 2. 고객 이메일 자동 발송     │
│    "시안을 발송했습니다"     │
│ 3. 대시보드 갱신             │
└─────────────────────────────┘
```

## ⑦ 상태별 색상 및 고객 알림

| 상태     | 배경색  | 고객 이메일 알림 |
|----------|---------|-----------------|
| 접수     | 연노랑  | ✕ (사업자만)    |
| 시안발송 | 연파랑  | ✅ 자동 발송    |
| 결제완료 | 연초록  | ✕               |
| 제작중   | 연주황  | ✕               |
| 배송중   | 연청록  | ✅ 자동 발송    |
| 완료     | 연회색  | ✅ 자동 발송    |
| 취소     | 연분홍  | ✕               |

고객 알림 상태를 바꾸려면 `CONFIG.NOTIFY_CUSTOMER_ON` 배열을 수정하세요.

## ⑧ 자주 묻는 오류

**Q. "권한이 없습니다" 오류**
→ `installTriggers()` 실행 시 구글 계정 권한 승인 필요. 팝업에서 "고급 → 주보라(안전하지 않음) 이동" 후 허용.

**Q. 이메일이 안 와요**
→ Gmail 할당량 확인 (무료 계정: 하루 100건). 또는 스팸 폴더 확인.

**Q. 주문번호가 중복돼요**
→ 폼 제출이 동시에 여러 건 들어올 경우 극히 드물게 발생. 발생 시 수동으로 수정.

**Q. 컬럼이 엇갈려요**
→ 폼 항목 순서와 `COLS` 상수를 다시 맞춰주세요 (⑤번 단계 참고).
